# 訓練記錄 Modal - 動作名稱移動與交換功能

## 需求

1. 動作名稱不顯示在 Modal Header，改為顯示在表單區塊最上方
2. Header 改為顯示菜單名稱（如「上肢日」）
3. 動作名稱右側新增交換按鈕，可選擇其他動作來記錄本次訓練
4. 交換為「僅本次」，不影響菜單配置

## UI 變更

### Modal Header

**Before:**

```
[取消]     高拉划船     [儲存]
```

**After:**

```
[取消]      上肢日      [儲存]
```

### 動作名稱區塊（新增，位於表單最上方）

```
┌─────────────────────────────────────┐
│  高拉划船                      [⇄]  │
└─────────────────────────────────────┘
```

### 動作選擇 Modal

```
┌─────────────────────────────────────┐
│  [✕]     選擇動作                   │
├─────────────────────────────────────┤
│  🔍 搜尋動作...                     │
├─────────────────────────────────────┤
│  ── 背部 ──                         │
│  高位下拉                           │
│  坐姿划船                           │
├─────────────────────────────────────┤
│  ── 其他動作 ──                     │
│  深蹲                               │
│  臥推                               │
└─────────────────────────────────────┘
```

**排序規則：**

- 相同部位的動作優先顯示（至少有一個部位重疊）
- 各組內按名稱排序

## 狀態管理

```typescript
// 新增 state
const [currentExercise, setCurrentExercise] = useState<{
  id: number;
  name: string;
} | null>(null);
const [showExercisePicker, setShowExercisePicker] = useState(false);
```

**流程：**

1. 開啟記錄 Modal → `currentExercise` 初始化為原動作
2. 點擊交換 → 開啟選擇 Modal
3. 選擇新動作 → 更新 `currentExercise`，重新載入最近紀錄
4. 儲存 → 使用 `currentExercise.id`
